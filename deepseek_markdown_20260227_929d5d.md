# 1. Производительность / отзывчивость UI

- Загрузка ленты новостей должна занимать ≤ 1 секунды при стабильном соединении (с учетом кеширования).
- Интерфейс реагирует на действия пользователя (фильтрация, поиск) без задержек > 150 мс.

---

# 2. Надёжность

- Приложение не должно падать при ошибках загрузки RSS-ленты (недоступность источника).
- Ошибки загрузки должны быть обработаны: пользователь видит заглушку или сообщение о проблеме с конкретным источником.

---

# 3. Хранение данных

- Список избранных статей сохраняется локально (`localStorage` или `IndexedDB`).
- Кеш новостей для ускорения загрузки при повторных посещениях.
- При повторном входе на сайт избранное и настройки фильтрации доступны пользователю.

---

# 4. Ограничения платформы

- Web-приложение должно работать в последних версиях Chrome, Edge, Firefox, Safari.
- Корректное отображение на мобильных устройствах (адаптивный дизайн).

---

# 5. Безопасность / приватность

- Так как авторизации нет, безопасность на уровне браузера (CORS). Проксирование запросов к RSS через серверную часть для обхода CORS-ограничений.
- Никакие пользовательские данные не собираются и не передаются третьим лицам.
