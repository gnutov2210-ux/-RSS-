# Этап 8. Тест-стратегия

**Файл:** `docs/08_test_strategy.md`

## 1. Unit-тесты
Проверяем изолированные функции и правила:
- Валидация `items_per_page` и `network_timeout_seconds`.
- Преобразование `RssEntryDTO -> NewsItem`.
- Очистка и нормализация заголовка/описания.
- Дедупликация новостей по `guid` или `link`.
- Сортировка новостей по дате публикации.
- Проверка TTL кэша.

## 2. Integration-тесты
Проверяем взаимодействие модулей:
- `FeedService + FeedparserRssProvider` на тестовой RSS‑ленте.
- `FeedService + SqliteFeedRepository` для сохранения и чтения снимка ленты.
- Обработка сетевых ошибок и переход к кэшу.
- Обработка пустой ленты.
- Обработка битого RSS / некорректного XML.

## 3. E2E / Acceptance
Проверяем сценарии глазами пользователя:
- Smoke‑тест: приложение запускается, главная страница отвечает HTTP 200.
- На главной странице показывается список новостей.
- При смене источника список обновляется.
- Поиск по слову уменьшает набор видимых новостей.
- По ссылке “Читать полностью” есть корректный URL.
- При ошибке сети показывается понятное сообщение.
