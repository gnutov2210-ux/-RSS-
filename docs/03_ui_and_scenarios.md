# Этап 3. UX-анализ: интерфейс, сценарии и ошибки

## 1. Список экранов / страниц
- Главная страница
- Страница новости
- Страница источников
- Страница сохранённых статей
- Страница настроек

## 2. Элементы на каждом экране

### Главная страница
- заголовок сайта;
- кнопка «Обновить новости»;
- поле поиска;
- выпадающий список источников;
- список карточек новостей;
- пагинация.

### Страница новости
- заголовок новости;
- источник;
- дата публикации;
- автор, если есть;
- краткое описание;
- кнопка перехода к оригиналу;
- кнопка сохранения.

### Страница источников
- список RSS-источников;
- статус источника (включён / выключен);
- форма добавления нового RSS-источника.

### Страница сохранённых статей
- список сохранённых новостей;
- кнопка удаления из сохранённых.

### Страница настроек
- количество новостей на странице;
- интервал обновления;
- кнопка сохранения настроек.

## 3. Состояния UI
- **Empty** — новостей ещё нет, пользователю предлагается выполнить первую загрузку.
- **Loading** — идёт обновление RSS-лент.
- **Success** — новости успешно загружены и отображаются.
- **Error** — произошла ошибка загрузки или добавления источника.

## 4. Пользовательские сценарии

### Сценарий 1. Загрузка новостей
1. Пользователь открывает главную страницу.
2. Нажимает кнопку «Обновить новости».
3. Приложение загружает RSS-ленты.
4. На экране появляется список новостей.

### Сценарий 2. Поиск новости
1. Пользователь вводит слово в строку поиска.
2. Нажимает кнопку применения фильтра.
3. Приложение показывает только новости, подходящие по заголовку.

### Сценарий 3. Сохранение статьи
1. Пользователь открывает список новостей.
2. Нажимает кнопку «Сохранить» у нужной статьи.
3. Новость попадает в раздел сохранённых.
4. Пользователь может позже открыть страницу «Сохранённые».

## 5. Типовые ошибки и тексты сообщений
- пустое название источника → **Введите название источника.**
- некорректный RSS URL → **Неверный RSS URL. Используйте http/https адрес.**
- RSS-лента недоступна → **RSS-лента недоступна или имеет неверный формат.**
- новость не найдена → **Запрашиваемая новость не найдена.**
- источник уже существует → **Не удалось добавить источник. Возможно, такой URL уже существует.**

## 6. CLI-вариант (служебные команды)

### Список команд
- `python app.py` — запуск веб-приложения
- `python app.py fetch` — загрузить новости из RSS-источников
- `python app.py sources list` — показать список RSS-источников
- `python app.py sources add "<name>" <rss_url>` — добавить новый RSS-источник
- `python app.py search "<query>"` — выполнить поиск по заголовкам новостей
- `python app.py saved list` — показать сохранённые статьи
- `python app.py saved add <news_id>` — сохранить статью
- `python app.py settings show` — показать текущие настройки
- `python app.py --help` — показать справку

### Примеры вывода

#### `python app.py sources list`
```text
[1] Habr (Технологии) - active
    https://habr.com/ru/rss/articles/?fl=ru
[2] Lenta.ru (Общее) - active
    https://lenta.ru/rss
```

#### `python app.py search "python"`
```text
- Python 3.13: что нового
  [Habr] https://habr.com/...
  id=ab12cd34
```

#### `python app.py settings show`
```text
items_per_page = 10
update_interval_minutes = 30
default_source_id =
```

### Состояния выполнения
- **Loading** — команда выполняет загрузку RSS-ленты.
- **Success** — данные успешно показаны в консоли.
- **Error** — выводится сообщение в едином формате: `Ошибка: <текст>`.
