# Этап 2. Нефункциональные требования

**Файл:** `docs/02_nfr.md`

## Производительность / отзывчивость
- Главная страница должна открываться не дольше чем за **2 секунды** при наличии кэшированных новостей.
- Обновление RSS по кнопке должно занимать не более **5 секунд** при стабильной сети и доступном источнике.
- Сервер не должен блокироваться на длительное время при ошибках внешнего RSS: таймаут сетевого запроса — не более **5 секунд**.

## Надёжность
- Приложение не должно аварийно завершаться из‑за одной невалидной RSS‑записи.
- Ошибки внешнего источника должны логироваться и показываться пользователю в понятном виде.
- Если обновление не удалось, приложение должно уметь показать последнюю успешно сохранённую ленту.

## Хранение данных
- Новости кэшируются локально в SQLite.
- Настройки приложения (например, последний источник и лимит количества новостей) сохраняются между перезапусками.
- Ссылки на новости должны храниться так, чтобы можно было отсеивать дубликаты.

## Ограничения платформы
- Целевая платформа: Windows / Linux / macOS.
- Достаточно локального запуска через Python без прав администратора.
- Требуется Python 3.11+ и список зависимостей в `requirements.txt`.

## Безопасность / приватность
- Секреты и служебные параметры не хранятся в репозитории в открытом виде.
- Если используются переменные окружения, файл `.env` должен быть добавлен в `.gitignore`.
- Приложение не хранит персональные данные пользователей.
- В HTML не должно быть небезопасной вставки неочищенного внешнего контента.
